## 1. 目的
把 TAITS 的工程開發分期（Phase 0–5）與 Canonical Flow（L1–L11）做可操作對照，避免每次新對話/新 Agent 流程漂移。

## 2. Phase 0–5 對照（S1）
- Phase 0：治理基線與口徑鎖定（本 S1 即 Phase 0 成果）
- Phase 1：資料面（L1–L3）工程落地與工件化
- Phase 2：分析/證據（L4–L5）落地與 Evidence Pack schema
- Phase 3：Regime/Gate（L6–L7）落地與否決權介面
- Phase 4：報告/裁決（L8–L10）落地：L9 報告模板 + L10 裁決模板 + 執行授權封套
- Phase 5：稽核回放（L11）落地：全層工件索引、hash ledger、回放工具鏈

## 3. 每個 Phase 的輸出物（最小集合）
- 代碼：對應模組
- 文件：對應 S1 分冊
- 工件：L11 可回放的 artifacts
