## 0. 本文件的存在理由（Why This File Exists）

TAITS 並非單一文件、單一策略、單一 AI 專案，
而是一個 **可長期演進、可審計、可交接、可擴充** 的完整系統。

在此架構下，**最常發生的失敗不是策略錯誤，而是文件失序**：

* 不知道哪一份才是最新
* 不知道誰能推翻誰
* 不知道哪些是制度、哪些只是說明
* 新 AI 讀錯文件就開始「自我發揮」

📌 **本文件的唯一目的**：

> 👉 成為 **TAITS 全專案「唯一且不可取代」的文件索引與治理入口**

## 1. 文件治理最高原則（Document Governance Constitution）

### 1.1 唯一索引原則（Single Index Rule）

* **本文件是 TAITS 唯一合法的文件總索引**
* 任何文件：

  * 未列於本索引
  * 或列名、路徑、定位不一致
    → 視為 **不存在 / 不可被 AI 採用**

---

### 1.2 治理階層不可顛倒（Hierarchy Non-Invertible）

文件之間 **嚴格存在不可逆的治理階層**：

| 層級 | 類型            | 是否可被推翻      |
| -- | ------------- | ----------- |
| A+ | 母體憲法 / AI 治理  | ❌ 不可        |
| A  | 架構 / 風控 / 索引  | ❌ 不可        |
| A- | 策略 / UI / Ops | ⚠️ 可被 A 級否決 |
| B  | 說明 / 教學       | ✅ 可更新       |

📌 **下位文件不得推翻上位文件，即使內容更新也不成立**

---

### 1.3 Only-Add 原則（不可縮水鐵律）

* **所有文件只能新增**
* 禁止：

  * 刪除段落
  * 合併概念導致資訊密度下降
  * 以「新版本」名義覆蓋舊設計思想

📌 若需淘汰內容，必須：

* 明確標示為「Deprecated」
* 保留全文
* 說明淘汰原因與替代位置
