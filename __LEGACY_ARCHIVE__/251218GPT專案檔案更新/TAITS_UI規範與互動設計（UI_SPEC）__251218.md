## 0. 文件定位（UI Is Control Surface）

本文件定義 **TAITS 所有使用者介面（UI）與互動設計的唯一規範**。  
在 TAITS 中：

> **UI 不是顯示層，  
> UI 是治理、風控與人類介入的主要控制面。**

📌 本文件的目標不是好看，而是：
- 防呆（Fail-Safe）
- 可解釋（Explainable）
- 可回放（Replayable）

---

## 1. UI 的核心設計原則（不可違反）

### 1.1 介面不做決策

- UI：
  - 不產生交易決策
  - 不自動觸發執行
- UI 只做：
  - 呈現
  - 提示
  - 確認
  - 否決

---

### 1.2 所見即證據（What You See Is Evidence）

- UI 顯示內容：
  - 必須對應實際資料
  - 必須可回溯
- 禁止：
  - 即時臨時計算但不留存

---

## 2. UI 分層架構（UI Layering）

TAITS UI 分為四層：

1. **Observation Layer**（觀察）
2. **Decision Support Layer**（決策輔助）
3. **Control Layer**（控制）
4. **Audit / Replay Layer**（審計回放）

📌 不同層級權限不同，不可混用。

---

## 3. 核心畫面類型與功能

### 3.1 市場觀察畫面（Observation）

- 市場狀態
- Regime 標示
- 流動性提示
- 異常警示

---

### 3.2 策略輔助畫面（Decision Support）

- 策略狀態（Enabled / Paused）
- 策略輸出摘要
- 信心權重
- 風控提示

📌 **不顯示下單按鈕。**

---

### 3.3 人類控制畫面（Control）

- 人類確認 / 否決
- 延後執行
- Kill Switch

📌 所有操作：
- 必須二次確認
- 必須記錄

---

### 3.4 回放與稽核畫面（Audit / Replay）

- 交易回放
- 決策鏈條
- 風控判斷

---

## 4. 交易單位差異顯示（零股 / 整股）

### 4.1 必須明確標示

- 每一策略、每一執行：
  - 清楚顯示交易單位
- 零股：
  - 顯示流動性風險
  - 顯示可能不成交

---

### 4.2 禁止混淆

- 禁止：
  - 零股顯示成整股績效
  - 混合單位不標示

---

## 5. 風控提示與否決顯示

### 5.1 風控提示等級

- Info（資訊）
- Warning（警告）
- Block（否決）

---

### 5.2 否決呈現方式

- 明確顯示：
  - 被哪一個 Gate 否決
  - 原因摘要
- 不允許：
  - 模糊訊息
  - 只顯示「失敗」

---

## 6. 人類操作流程（Human-in-the-Loop UI）

### 6.1 必須二次確認的行為

- 啟用新策略
- 解除風控限制
- 進入 Live-Ready

---

### 6.2 操作紀錄

- 操作者
- 時間
- 行為
- 結果

---

## 7. UI 與 Execution 的邊界

- UI：
  - 發出「授權意圖」
- Execution：
  - 只接受已授權命令

📌 UI 不得直接呼叫下單 API。

---

## 8. 失敗、異常與防呆設計

- API 異常：
  - UI 顯示不可交易
- 資料異常：
  - UI 顯示資料不可用
- 系統異常：
  - UI 禁止操作

---

> 而是讓你「不會犯錯」。**
