## 0. 專案定位（請先完整閱讀）

**TAITS（Taiwan Alpha Intelligence Trading System）** 是一套：

- 專為 **台灣市場（TWSE / TPEX）** 設計
- 可長期演進、可審計、可回放
- 以 **Regime（市場狀態）與 Risk / Compliance（風控 / 合規）** 為最高優先
- 以 **文件治理（Docs > Code）** 為核心運作方式
- **AI 輔助決策，但 AI 永遠不是唯一真理**

TAITS 不是教學 Demo、不是單一策略、不是只為回測而生，  
而是一套 **可落地、可治理、可擴充** 的系統母體。

---

## 1. 核心設計哲學（不可違反）

以下原則對 **所有人與所有 AI** 具有最高約束力：

- **策略 ≠ 下單**
- **Agent ≠ 策略**
- **AI ≠ 唯一真理**
- **Regime（市場狀態）高於任何單一訊號**
- **Risk / Compliance 具最高否決權（Veto Power）**
- **官方來源優先，易變動細節不得寫死，只能引用**

任何違反以上原則的設計、實作、建議，  
在 TAITS 中 **一律視為無效**。

## 2. 現階段 Scope（避免新對話跑偏）

### 2.1 市場與產品

- 市場：
  - TWSE（臺灣證券交易所）
  - TPEX（櫃買中心）
- 產品：
  - 股票（Stock｜股票）

### 2.2 交易單位（Trade Unit）

- **核心優先：零股（Odd Lot｜零股）**
- 設計必須同時考慮：
  - 零股策略
  - 整股策略
  - 混合策略（預留，是否啟用由治理決定）
- 原因：
  - 預算限制
  - 零股流動性差異
  - 實務成交與風控可行性

## 3. 券商與平台策略

### 3.1 主要券商 API

- **富邦證券 TradeAPI（Fubon TradeAPI）**
  - 為現階段主要設計與實作目標
  - 所有 Execution 設計以其能力與限制為準

### 3.2 明確排除（現階段）

- **XQ 平台**
  - 原因：
    - 平台封閉
    - 無法進行零股程式交易
  - 規則：
    - 現階段不得與 Execution 層產生任何依賴
    - 未來若要納入，必須新增「解鎖治理文件」

## 5. 系統組成總覽（高層視角）

TAITS 由以下核心層級構成：

1. **治理層（Governance）**
   - 文件索引
   - 版本控管
   - GitHub 存檔規則
   - AI 行為規範

2. **架構層（Architecture）**
   - 主架構鐵律
   - 全系統模組分層
   - 資料流 / 決策流

3. **資料層（Data）**
   - 官方資料優先
   - 多層備援
   - 可回放、可審計

4. **策略層（Strategy）**
   - 多策略並存
   - 以 Regime 為結構
   - 策略本身不直接下單

5. **風控與合規層（Risk & Compliance）**
   - 最高否決權
   - 即時與事後審計

6. **執行層（Execution）**
   - 僅在通過風控後執行
   - 必須保留人類介入權

7. **維運與介面（Ops / UI）**
   - 監控
   - 回溯
   - 防呆顯示
