# TAITS_Beginner_Teaching_Rules.md
# TAITS — 小白 / 新接手教學模式（Beginner Teaching Mode）

版本：Master  
文件角色：**Onboarding 強制規範**

---

## 0. 何時必須啟用本模式

當使用者或 AI 明確表示以下任一狀況：

- 「我不會操作」
- 「請一步一步教我」
- 「我是新手 / 小白」
- 「我第一次接觸 TAITS」

👉 **必須啟用 Beginner Teaching Mode**

---

## 1. 教學基本原則（強制）

1. 一次只給一個操作動作  
2. 等待對方回報完成後再繼續  
3. 避免專業術語（必要時附白話解釋）  
4. 每一步都要有「子目標」  
5. 不一次講太多內容  
6. 教學必須導向「成功完成任務」

---

## 2. 教學語氣與方式

必須使用：

- 「你現在請做這個」
- 「你會看到這個畫面」
- 「如果看到 X，請告訴我」

❌ 禁止一次給整段流程

---

## 3. 出錯時的處理方式

1. 請使用者描述畫面
2. 不假設對方錯在哪
3. 根據實際環境（OS / 工具）調整說明
4. 必要時要求截圖或貼訊息

---

## 4. 程式碼提供規範

- 必須提供 **完整可複製版本**
- 不給 diff
- 不給片段
- 不要求使用者自行補齊

---

## 5. 教學完成定義

以下條件成立，才算完成：

- 使用者已成功完成目標動作
- 使用者能描述「剛剛做了什麼」
- 系統狀態正確（未誤觸 Live）

---

## 6. 一句話總結

> **Beginner Mode 的目標不是快，  
> 而是「不出錯」。**

# TAITS — GitHub 儲存檔案規範（Mandatory Save Rules）

## 0. 規範適用範圍

本規範適用於：

- 人工上傳 GitHub
- AI 協助產生並存檔
- 文件治理、版本管理

## 1. 觸發條件

只要出現以下語句之一：

- 「幫我存成專案文件」
- 「上傳到 GitHub」
- 「這份要放進 TAITS」

👉 **必須遵守本規範**

---

## 2. 強制提供四項資訊（缺一不可）

任何存檔回覆 **必須同時提供**：

1. **檔案名稱**  
2. **完整 GitHub 儲存路徑**  
3. **檔案用途說明**  
4. **完整檔案內容（可直接貼上）**

❌ 缺任一項，不得宣稱「已完成」

---

## 3. 內容品質要求

- 不可給摘要
- 不可給 diff
- 不可要求使用者自行補齊
- 必須可直接落盤

---

## 4. 文件優先順序

- 文件治理 > 程式碼
- 架構文件 > 實作文件
- Risk / Compliance > 其他文件

---

## 5. 一句話總結

> **沒有完整四項，  
> 就不算存檔完成。**
