# 📘 TAITS_Shadow_Mode_Evaluation_Metrics.md
## TAITS 影子模式評估指標（Go / No-Go Gate）

---

## 0. 文件定位（必讀）

本文件定義 TAITS 在 **SHADOW（影子模式）** 下的
- 量化指標
- 行為檢查
- 風控一致性
- 升級門檻

> 未通過本文件標準者：
> - 禁止進入任何實盤模式
> - 不得以「感覺不錯」或「短期賺錢」作為理由升級

---

## 1. SHADOW 最低運行要求（硬門檻）

### 1.1 最低樣本數

必須同時滿足：

- ⬜ 交易日數 ≥ **30 個交易日**
- ⬜ 產生決策日 ≥ **20 日**
- ⬜ TRADE / WAIT / NO_TRADE 三類皆出現

📌 **樣本不足 → 不評估**

---

### 1.2 市場狀態覆蓋

SHADOW 期間必須涵蓋至少兩種不同 Regime：

- ⬜ TREND
- ⬜ RANGE
- ⬜ HIGH_VOL / EVENT（可為短期）

📌 **只跑單一順風狀態 → 評估無效**

---

## 2. 決策品質指標（比報酬重要）

### 2.1 NO_TRADE 比例（健康度）

- 目標範圍：**20% ～ 50%**
- < 10%：過度交易（Fail）
- > 60%：過度保守（需檢視）

---

### 2.2 Gate / MR 否決一致性

檢查以下行為：

- ⬜ MR-3 時是否 100% NO_TRADE
- ⬜ Gate FAIL 時是否從未產生 TRADE
- ⬜ MR-2 時是否無加碼行為

📌 **任一違反 → 直接 Fail**

---

### 2.3 決策自洽性（Stability）

- 同一市場條件下，Decision 不應頻繁翻轉
- 不得出現：
  - 上午 TRADE → 下午 NO_TRADE（無新證據）
  - 前後邏輯矛盾

📌 **頻繁翻轉 = 架構不穩**

---

## 3. 風險行為指標（紅線）

### 3.1 最大模擬回撤（MDD）

- 允許範圍：**可承受實盤回撤 × 0.5**
- 超過 → 不升級

📌 **SHADOW 回撤比實盤容忍更嚴格**

---

### 3.2 連續錯誤反應

- 是否在連續錯誤後：
  - 降權
  - WAIT
  - NO_TRADE

📌 **若仍持續 TRADE → Fail**

---

## 4. 報酬指標（非必要，但參考）

> 報酬只作為「輔助」，不是通關條件。

可觀察但不硬性要求：

- 勝率 ≥ 40%
- Profit Factor ≥ 1.1
- 平均盈虧比 ≥ 1.2

📌 **短期虧損 ≠ 失敗**

---

## 5. 操作一致性檢查（人 / 系統）

### 5.1 人為干預一致性

- ⬜ 人為判斷 ≈ 系統判斷
- ⬜ 無頻繁「覺得應該做」但系統不做

📌 **人腦常常是風險來源**

---

### 5.2 流程遵循率

- 是否每個交易日都完整走：
  - Decision Checklist
  - Playbook

📌 **跳步一次 = 記錄一次警告**

---

## 6. 升級決策矩陣（Go / No-Go）

| 條件 | 狀態 |
|---|---|
| 最低運行要求 | ⬜ Pass / ⬜ Fail |
| Gate / MR 一致性 | ⬜ Pass / ⬜ Fail |
| 風險行為 | ⬜ Pass / ⬜ Fail |
| 決策穩定性 | ⬜ Pass / ⬜ Fail |

### 結論（只能選一個）：
- 🟢 **GO：可升級至 LIVE_RESTRICTED**
- 🟡 **HOLD：延長 SHADOW 觀察**
- 🔴 **NO-GO：回到策略 / 風控調整**

---

## 7. 升級後的第一階段限制（強制）

即使 GO，也必須：

- 倉位 ≤ 正式規模的 **25%**
- 禁止加碼
- 僅限單一標的或零股
- 持續監控 MR / Gate

---

## 8. 官方準則（不可刪）

> **TAITS 不是因為「看起來會賺」才進實盤，  
> 而是因為「即使不賺，也不會亂做」才進實盤。**

---

## 9. 文件狀態

- ✔ SHADOW → LIVE 唯一門檻
- ✔ 可審計、可回放
- ✔ 防止衝動升級
- ✔ 與 Execution Safety Locks 完全相容
